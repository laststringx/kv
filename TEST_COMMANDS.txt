## Expected Outcomes

✅ All PUT commands should show: `✓ Stored: <key> = <value>`  
✅ All GET commands should return the stored value or "Key not found"  
✅ All DELETE commands should show: `✓ Deleted: <key>`  
✅ CLEAR should show: `✓ All data cleared`  
✅ STATUS should display store information  
✅ After EXIT and restart, data should be recovered from WAL

## How to Use This File

**Option 1 - Manual Testing:**
1. Open `kvds.bat` 
2. Copy commands one by one from this file
3. Paste into CLI and press Enter
4. Verify output matches expected results

**Option 2 - Quick Test:**
1. Copy the entire "Quick Copy-Paste Block"
2. Paste into CLI
3. Watch all commands execute

**Option 3 - Crash Recovery Test:**
1. Run Session 1 commands
2. Exit CLI
3. Restart CLI
4. Run Session 2 commands
5. Verify data persisted

---

## Notes

- Commands are case-insensitive (PUT = put = Put)
- Values with spaces won't work - use underscores
- Pipe character `|` is not allowed in keys or values
- Each test case is independent
- WAL file `kvds-cli.log` stores all operations
